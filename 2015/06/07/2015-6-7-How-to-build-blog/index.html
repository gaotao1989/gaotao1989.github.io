利用Github和Hexo快速搭建博客系统

很早就想搭建技术博客，终于在折腾了一天后利用Hexo与Github搭建好了免费博客系统，这篇文章将记录搭建过程。

前言

萌生开设技术博客的想法已经很久，也注册过很多博客帐号，如新浪，CSDN等等，最近还在新浪SAE上利用Wordpress搭建过博客系统，然而一直没有找到满意的方式。个人对博客系统有几点要求：1）首先必须免费或者开销非常低。像土豪们租用服务器搭建独立博客，不是我想要的。2）其次管理维护必须非常简单，我可不想花费太多时间于博客系统本身。3）最后博客的内容应该很容易迁移。技术博客更多的是学习总结，所以我希望对这些知识具有控制权。利用Hexo与Github基本能够满足我的这些需求。

什么是Hexo与Github？

Hexo是一个基于nodejs的静态博客网站生成器，作者是来自台湾的Tommy Chen。其中Node.js是一套用来编写高性能网络服务器的JavaScript工具包。Hexo的特点包括：快速、简单; 支持Markdown（Markdown是一种可以使用普通文本编辑器编写的标记语言，编写博客非常方便）;只需要一条命令就可以部署到GitHub Pages上;另外，Hexo也可以兼容Windows, Mac与Linux系统。

Github是一个开源代码库及版本控制系统，Github支持Pages用于展示个人主页或项目主页，利用Github搭建博客的原理就是把博客系统挂到个人主页或者项目主页上。

安装Hexo

  • 安装nodejs与npm
    我使用的操作系统是ubuntu 12.04，首先需要安装nodejs与npm，安装命令如下：

    sudo apt-get install nodejs
sudo apt-get install npm

    如果这样安装得nodejs不满足版本要求，也可以自行下载源码包进行安装。

  • 安装Hexo
    npm安装好后就可以通过npm安装hexo，命令如下：

    sudo npm install -g hexo

  • 初始化Hexo
    安装好hexo后，就可以创建一个博客目录，然后在该目录下输入如下命令，注意执行初始化命令后需要利用npm安装相应的依赖程序：

    hexo init
sudo npm install

本地测试

Hexo本地安装完成后就可以进行本地测试了，首先需要用hexo generate命令生成网页，然后利用hexo server命令启动本地服务器：

hexo generate
hexo server

部署到github

部署到github首先需要注册github帐号，然后创建repositories，并在master中创建Pages，然后修改hexo中的配置文件_config.yml如下所示：

type: git
repository: https://github.com/gaotao1989/gaotao1989.github.io
branch: master

需要注意得是早期有些版本中type应该写成github，修改之后需要运行如下命令：

sudo npm install hexo-deployer-git --save

然后就可以通过如下命令发布到Github的Pages上：

hexo generate
hexo deploy

更换主题

互联网上提供有多种hexo主题，可以通过这里查看。
选好自己喜欢的主题之后，通过如下命令下载到本地目录，然后在_config.yml文件中选用相应得主题即可。

git clone <repository> themes/<theme-name>

我使用如下命令获得jacman主题：

git clone https://github.com/wuchong/jacman.git themes/jacman

写第一篇文章

配置好博客的外观就可以开始写博客了，通过如下命令新建文章：

hexo new <name>

新建文章实际上是在source/_posts目录下新建一个.md文件，该文件采用Markdown语法编写。

title: 第一篇文章         # 文章标题
date: 2015-06-07 20:30:00 #文章生成时间
categories: example       # 分类
tags: [tag1,tag2,tag3]    # 文章标签
description:           # 文章摘要
---
