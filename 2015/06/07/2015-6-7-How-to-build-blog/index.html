利用Github和Hexo快速搭建博客系统

By Alvin

发表于 2015-06-07

想开设自己的个人博客，又不想花费太多精力对博客系统进行维护。本文即介绍如何利用Hexo与Github搭建简单的个人博客系统。

前言

一直以来想开通个人技术博客，这不仅有助于进行知识管理，还能够督促自己多思考。开通个人博客可以有多种选择：1）在新浪、科学网等网站上开通。在这些大型网站上开通博客，可以免除系统维护的麻烦，但是这些文档的可移植性不强; 2) 自己购买服务器或云服务，利用一些开源博客系统如wordpress搭建自己的独立博客。这种方法需要管理者自己进行一些维护工作。
我开设博客的主要目的是进行知识管理，因而希望能够把主要精力集中在撰写总结文章上，而不是花费在繁琐的管理工作上。在对比了多种方案后，我选择了Hexo+Github方案。Hexo是一个静态博客系统框架，用户通过Markdown语法进行博客的编写，利用hexo就可以生成相应的网页文件。同时，我们可以利用Github提供的免费Pages部署博客系统。

安装Hexo

我使用的操作系统是ubuntu 12.04，首先需要安装nodejs与npm，然后通过npm安装hexo，然后就可以通过hexo创建博客系统，最后再部署到github上，即大功告成。

  • 安装node
    ubuntu下安装node比较简单，通过apt-get安装就行，但是这样安装得版本不一定满足要求，如果需要安装最新版本，可以从源代码采用经典的./configure && make && make install安装

    sudo apt-get install nodejs

  • 安装npm
    ubuntu下安装npm比较简答，命令如下：

    sudo apt-get install npm

  • 安装hexo
    npm安装好后就可以通过npm安装hexo，命令如下：

    sudo npm install -g hexo

  • 初始化hexo
    安装好hexo后，就可以创建一个博客目录，然后在该目录下输入如下命令：

    hexo init
sudo npm install

本地测试

hexo generate
hexo server

部署到github

部署到github首先需要注册github帐号，然后创建repositories，并在master中创建Pages，然后修改hexo中的配置文件_config.yml如下所示：

type: git
repository: https://github.com/gaotao1989/gaotao1989.github.io
branch: master

早期有些版本中type应该写成github，修改之后需要运行如下命令：

sudo npm install hexo-deployer-git --save

然后就可以通过如下命令发布到Github的Pages上：

hexo generate
hexo deploy

更换主题

互联网上提供有多种hexo主题，可以通过这里查看。
选好自己喜欢的主题之后，通过如下命令下载到本地目录，然后在_config.yml文件中选用相应得主题即可。

git clone <repository> themes/<theme-name>

我使用如下命令获得jacman主题：
git clone https://github.com/wuchong/jacman.git themes/jacman

写第一篇文章

配置好博客的外观就可以开始写博客了，通过如下命令新建文章：

hexo new <name>

新建文章实际上是在source/_posts目录下新建一个.md文件，该文件采用Markdown语法编写。

title: 第一篇文章         # 文章标题
date: 2015-06-07 20:30:00 #文章生成时间
categories: example       # 分类
tags: [tag1,tag2,tag3]    # 文章标签
description:           # 文章摘要
---
