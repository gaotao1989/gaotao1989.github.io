本文记录Git的基本使用方法。（持续更新中…）

Git简介

在管理代码时，我们通常希望清楚得知道当前代码相对于先前的版本有哪些改动，并且希望在必要时很快回滚到先前的状态，同时我们某些时候可能需要基于相同的代码分别开发不同的功能，这些即是所谓的版本控制。版本控制在多人协同开发相同的软件时显得更加重要。

早期，人们主要利用CVS或SVN等进行版本控制（SVN现在仍被很多人使用），这类版本控制系统被称为集中式版本控制系统。在集中式版本控制系统时，存在一个中心服务器，所有人的修改都提交到中心服务器。这样带来的问题时一旦中心服务器不可用，所有人都无法提交修改。

和集中式版本控制系统不同的是，Git采用分布式设计的思想，即每个客户端都拥有自己的版本库，本地版本库可以和任意其它版本库进行合并操作。在实际的Git开发中，通常也有一个中心服务器，然而即使中心服务器不可用，开发者也可以互相交换修改，这样就避免了集中式版本控制系统的弊端。

安装Git

Git在Linux, Window, Mac等系统上都有客户端，用户可以点击这里查看。在ubuntu上安装很简单，命令如下：

sudo apt-get install git

使用Git

  • 创建项目仓库
    有两种方法初始化版本库，第一种方法是初始化空的版本库：

    git init

另一种方法是从远程库克隆一个版本库：

git clone <link>

项目仓库创建好之后，在仓库目录下存在一个.git目录，该目录即包含了整个git项目的数据。

  • 文件检入与检出
    Git仓库模型大致分成三个工作区域，分别为工作目录(Working Directory)，暂存区域(Stage)以及本地仓库(History) 。Git文件在三个工作区域检入与检出的命令如下所示 :

    git add files: 把文件拷贝到暂存区域 (工作目录->暂存区域)
git commit: 把暂存区域的文件提交到本地仓库(暂存区域->本地仓库)
git reset -- files: 撤销暂存区域文件
git checkout -- files:  把文件从暂存区域覆盖本地文件，丢弃本地修改(暂存区域->工作目录)
git commit (files | -a): 先将文件复制到暂存区域，然后再提交到本地仓库(工作目录->暂存区域->本地仓库)
git checkout HEAD -- files: 回滚到最后一次提交，即用本地仓库中的最后一次提交覆盖工作目录的内容

为查看工作目录，暂存区域以及本地仓库的文件有哪些不同，可以利用 git diff命令：

git diff: 查看工作目录与暂存区域文件的不同
git diff --cached: 查看暂存文件与上次提交的本地仓库快照之间的不同
git diff HEAD: 查看工作目录文件与上次提交的本地仓库快照之间的不同
git diff <index1> <index2>: 查看两个不同快照之间的差别

掌握了这些基本的使用，就可以利用Git对项目进行基本的管理。下面介绍一些更高级的特性。

分支管理

Git中存在一个默认分支称为master，可以创建更多分支，命令如下所示：

git branch <branch name>

在创建分支之后，可以利用checkout命令检出分支代码，需要指出的是HEAD指针总是指向当前分支，因而当切换到分支时，HEAD指针也相应得进行切换。

git checkout <branch name>

分支在一定情况下需要合并，合并命令如下所示（分支合并时可能存在冲突，这时即需要人工进行冲突处理）：

git merge <branch name> : 和并分支，提交历史不是线性的
git rebase <branch name>： 衍合分支，保证提交历史是线性的

分支和并之后可以删除分支，命令如下：

git branch -d <branch name>

标签管理

Git支持打标签，标签命令如下所示：

git tag -a v0.1.2 -m "Release version 0.1.2" ： 创建标签
git tag –l ： 查看标签
git show v0.1.2： 查看标签等详细信息
git tag -d v0.1.2: 删除标签
git push origin v0.1.2：把本地标签上传到远端仓库中
git push origin --tags: 上传所有本地新增标签

远程仓库

Git是分布式版本控制系统，要实现本地仓库和远程仓库之间的数据交换，就需要利用远程仓库命令：

git remote：查看远程仓库
git git push origin master: 把本地仓库的数据推送到远程仓库
git fetch [remote-name]: 获得本地仓库中没有的数据
git pull [remote-name]： 将远程分支自动合并到当前分支，相当于fetch+merge命令
